Vibes

Overview

In this project, we need to design and implement an application that uses multiple mobile device features in an interesting and useful way. My teammates and I decided to create a music player app with a variety of functions to enhance the user experience.


Design focus

We intend to build a music player with a variety of functions to enhance user experience. The application will feature user-friendly UI supporting minor user preference settings and a one-time login functionality upon launching the application.

Other Info:

Team size: 2
(Jeffrey Chen & Ken Hsueh)

Duration: Oct 2020 - Dec 2020 (6 weeks)

Tools: Android Studio, Figma

My role: App developer & User Interface Designer

Process Analysis + Challenges

View in details

Create application proposal

My role in this stage:

  • Come up with the functionalities of our app with my teammate.
  • Create the application flow.
  • Draw low-fidelity interface sketches.
  • Create low-fidelity Prototypes with Figma.

Challenges:

  • This project started in the middle of the course which we only learn some basic of Android Studio.
  • We are not sure what Android Studio can do and it is hard to determine the scope of our project.

How we solve:

First, we came up with the idea of building a music player. We started to do some research about creating a music player with Android Studio. Additionally, we asked for some previous student projects from our professor to get a better idea of the expectation of this project.

Building the basic functionality of the applicaiton

My role in this stage:

  • Build the login, register and playlist interface.
  • Implement a SQLite database to store the account information.
  • Make the rotation animation when playing the music.
  • Implement the accelerometer sensor to detect shaking events on usersâ€™ phone and it will skip to the next song.

Challenges:

  • We had trouble in figuring out the implementation of GPS functionality specifically in utilizing the location to recommend an exclusive playlist.
  • It is not applicable as we would have to set up individual lists for all the countries across the world (195 lists).

How we solve:

Our team decided to use a region-based solution that only have 7 continents (Asia, Africa, North America, South America, Antarctica, Europe, and Australia). In other words, we can create 7 different music lists that represent each continent.

Final version + Reflection

Overall, this project is successful. Although my teammate was in a different time zone due to the pandemic, we still managed to complete most of the functionalities of the application. The function that allows users to create their personal playlists was not implemented because we ran out of time to explore it. This also taught us a lesson about the importance of time management.

Some Takeaways from this project:

  • It is important to have a confirmation message when users are going to make some changes (such as deleting an account). This can stop the errors before they happen.
  • Good time management should be set before you start a project. Plan ahead and set a time limit to complete each of the tasks in a project can prevent having an inefficient workflow.
